# Glossary

- **Alertmanager**: component that receives alerts from Prometheus, groups them, inhibits duplicates and routes notifications.
- **Charm**: Juju operator package that encodes operational lifecycle knowledge.
- **Day 1 operations**: initial deployment and configuration work.
- **Day 2 operations**: ongoing operations such as upgrades, scaling, backup, incident response and tuning.
- **eBPF**: Linux kernel technology that enables safe programmable hooks for observability, networking and security.
- **Grafana**: visualization and correlation layer for metrics, logs, traces and alerts.
- **Juju model**: operational workspace for charms, applications, relations and configuration.
- **Loki**: log aggregation backend optimized around labels and LogQL.
- **Mimir**: horizontally scalable, multi-tenant metrics backend compatible with Prometheus-style metrics.
- **OpenTelemetry**: vendor-neutral APIs, SDKs and Collector for telemetry instrumentation and transport.
- **Parca**: continuous profiling system often using eBPF to sample CPU profiles.
- **Prometheus**: metrics scraper, time-series database, PromQL engine and alert rule evaluator.
- **Tempo**: distributed tracing backend for storing and querying traces.
